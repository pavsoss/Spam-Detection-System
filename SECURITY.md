# Security Policy

## Supported Versions

Currently, only the latest version of the **Spam Detection System** is actively maintained and receives security updates.

| Version | Supported |
| ------- | --------- |
| Latest  | ✅         |

## Reporting a Vulnerability

If you discover a security vulnerability in the Spam Detection System, please **do not create a public GitHub issue**.

Instead, report it privately by creating a **GitHub Security Advisory** (if enabled) or by contacting the project maintainers through GitHub.

When submitting a report, please include:

* A clear description of the vulnerability.
* Steps to reproduce the issue.
* The potential security impact.
* Screenshots, logs, or a proof of concept (if applicable).

We aim to acknowledge security reports within **2–5 business days** and will investigate all valid reports as quickly as possible.

Thank you for helping keep the Spam Detection System secure for everyone.
